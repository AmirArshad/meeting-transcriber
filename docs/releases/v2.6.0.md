# AvaNevis v2.6.0

Back-to-back recording and background transcription queue — record the next meeting as soon as the previous file is saved.

## What's New

### Sequential recording (background transcription queue)

- **Start again after save** — after Stop finishes encode and the meeting is persisted as `pending`, capture returns to Ready. You can record Meeting 2 while Meeting 1 still transcribes. Encode still blocks Start; Whisper does not.
- **Home Activity list** — replaces the single post-recording transcript panel. Rows show Queued / Transcribing / Identifying speakers / Waiting for GPU or model setup / Ready / Failed with correct meeting identity and duration.
- **Main-owned jobs** — each finished recording becomes a per-meeting composite job (transcribe → optional diarization → persist) on the existing local AI compute queue, so meetings stay FIFO and cannot interleave mid-pipeline.
- **Cancel transcription** — cancel a queued or active Activity job without quitting; cancelled work becomes durable `failed` (“Cancelled by user”) and stays retryable from History.
- **Discard recording** — Cancel/Discard during an in-progress capture throws away the session: no meeting, no queue entry, no recovery resurrection on relaunch.
- **Quit honesty** — quitting terminates in-flight transcription jobs; audio and `pending` metadata survive so work can finish next launch. Dialog copy: recordings will finish transcribing next time you open AvaNevis.
- **Resume pending** — auto-resume durable `pending` transcriptions after recordings maintenance completes (with per-row cancel); failed jobs never auto-resume.
- **Activity hygiene** — rename and delete from Home Activity (same metadata paths as History); completion toasts; first-run tip that you can record again while transcription runs; soft queue-depth guidance.
- **GPU / model admission** — Whisper preload and GPU runtime install/repair admit between compute jobs on a shared resource queue so pip and inference do not race loaded CUDA DLLs.

### Fixes included

- Activity Rename uses an inline editor (Electron does not support `window.prompt`).
- macOS packaged builds bundle `soxr` for shared spool finalization.
- Pre-release adversarial hardening for recorder cancel settlement, compute wall-clock timeouts, queue snapshot ordering, and Windows capture-session lock cleanup.

## Platform Support

- Windows 10/11 x64
- macOS 13+ on Apple Silicon

## Notes

- Only one live recording at a time; overlapping capture is still unsupported.
- Mic and desktop tracks remain separate until post-recording mix.
- Whisper models still download on first use unless already cached.
- GPU acceleration is used automatically when supported (CUDA on Windows, Metal on Apple Silicon).
- Optional speaker identification and meeting summaries remain explicit local AI add-ons.

## Privacy

- Zero telemetry
- No cloud transcription
- No account required
