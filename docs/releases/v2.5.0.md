# AvaNevis v2.5.0

Recording awareness (Release 1) and long-recording safety (Release 2).

## What's New

### Recording awareness (Release 1)

- **Always-visible recording status** — in-app pill and `H:MM:SS` elapsed clock across Record, History, and Settings; authoritative `starting` / `recording` / `stopping` / `idle` lifecycle from the main process.
- **OS presence indicators** — macOS static saturated menu-bar icon + `REC` title and Dock badge; Windows taskbar overlay while the window is minimized during recording.
- **Hourly native reminders** — best-effort notifications while a recording is active (never auto-stop on duration); click-to-open uses retained notification objects (Electron has no Action Center cold-activation API).
- **Single-instance focus** — launching AvaNevis again focuses the existing window instead of starting a second copy.
- **Recording-aware close copy** — Windows: keep recording minimized; macOS: keep running from the menu bar; graceful quit/save path unchanged.
- **Discoverability metadata** — improved Start Menu / Spotlight descriptions for searches like “meeting” or “transcriber” without changing `productName`, installer identity, or `userData` paths.

### Long-recording safety (Release 2)

- **Durable capture spools** — mic and desktop audio spill to `{stem}.capture/` on disk during recording (not whole-session RAM). Spool files are never scan-imported as meetings.
- **Bounded stop finalization** — multi-pass normalize/mix/encode with structured stop stages (`Finishing recording…`) instead of allocating whole-recording arrays in memory.
- **Interrupted-session recovery** — if the app or recorder dies mid-meeting, the next launch offers **Recover Now** / **Later**, keeps capture files on dismiss/failure, and shows a persistent banner for deferred sessions.
- **Low-disk warnings** — Node `statfs` free-space checks; warning below 10 GB and critical below 2 GB; periodic checks while recording; never auto-stop solely because a threshold was crossed.
- **Transcription runtime metadata** — completed meetings record the resolved Whisper device/compute type (including MLX `metal` → `mps` normalization for persistence).

### Dependency / packaging hygiene (included)

- `adm-zip` 0.6.0 (zip extraction hardening, including CVE-2026-39244).
- Runtime pin bumps: `filelock`, `certifi`, `protobuf`, `regex`.
- Windows packaged `setuptools` 83.0.0 (pip-audit clear); macOS remains on 81 for torch install resolution (torch is still removed from the packaged app after `pip install`).

## Platform Support

- Windows 10/11 x64
- macOS 13+ on Apple Silicon

## Notes

- Mic and desktop tracks remain separate until post-recording mix (no real-time mixing redesign).
- Whisper models still download on first use unless already cached.
- GPU acceleration is used automatically when supported (CUDA on Windows, Metal on Apple Silicon).
- Optional speaker identification and meeting summaries remain explicit local AI add-ons.

## Privacy

- Zero telemetry
- No cloud transcription
- No account required
